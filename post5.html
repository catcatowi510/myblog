Git & GitHub - Quản Lý Code

Hướng dẫn sử dụng Git và GitHub để quản lý mã nguồn hiệu quả và làm việc nhóm

Git Là Gì?

Git là một hệ thống quản lý phiên bản phân tán (Distributed Version Control System) được phát triển bởi Linus Torvalds. Git giúp lập trình viên theo dõi các thay đổi trong mã nguồn, làm việc nhóm hiệu quả và quản lý nhiều phiên bản của dự án.

GitHub Là Gì?

GitHub là một nền tảng lưu trữ mã nguồn trực tuyến sử dụng Git. GitHub cung cấp giao diện web thân thiện, tính năng collaboration mạnh mẽ và là nơi lưu trữ hàng triệu dự án open source trên thế giới.

Cài Đặt Git

# Windows - Download từ git-scm.com

# macOS
brew install git

# Linux (Ubuntu/Debian)
sudo apt-get install git

# Kiểm tra version
git --version

# Config thông tin
git config --global user.name "Vũ Thùy Linh"
git config --global user.email "vuthuylinh@email.com"

Các Lệnh Git Cơ Bản

Khởi Tạo Repository

# Tạo repo mới
git init

# Clone repo có sẵn
git clone https://github.com/username/repo-name.git

Làm Việc Với Files

# Kiểm tra trạng thái
git status

# Thêm file vào staging area
git add filename.txt
git add .  # Thêm tất cả files

# Commit changes
git commit -m "Add new feature"

# Xem lịch sử commits
git log
git log --oneline  # Xem ngắn gọn

Branches

# Xem danh sách branches
git branch

# Tạo branch mới
git branch feature-name

# Chuyển sang branch khác
git checkout feature-name
# Hoặc (Git 2.23+)
git switch feature-name

# Tạo và chuyển sang branch mới
git checkout -b feature-name
# Hoặc
git switch -c feature-name

# Merge branch
git checkout main
git merge feature-name

# Xóa branch
git branch -d feature-name

Remote Repository

# Thêm remote
git remote add origin https://github.com/username/repo.git

# Xem remote
git remote -v

# Push code lên GitHub
git push origin main
git push -u origin main  # -u để set upstream

# Pull code về
git pull origin main

# Fetch (chỉ download, không merge)
git fetch origin

Git Workflow Cơ Bản

# 1. Tạo/chuyển sang branch mới
git checkout -b feature/new-feature

# 2. Làm việc và commit
git add .
git commit -m "Implement new feature"

# 3. Push lên GitHub
git push origin feature/new-feature

# 4. Tạo Pull Request trên GitHub

# 5. Sau khi merge, update main branch
git checkout main
git pull origin main

# 6. Xóa branch đã merge
git branch -d feature/new-feature

Xử Lý Conflicts

Khi merge branches, đôi khi xảy ra conflicts:

# Khi có conflict
git merge feature-branch
# Auto-merging file.txt
# CONFLICT (content): Merge conflict in file.txt

# Mở file và sửa conflicts
<<<<<<< HEAD
Code của branch hiện tại
=======
Code từ branch merge vào
>>>>>>> feature-branch

# Sau khi sửa xong
git add file.txt
git commit -m "Resolve merge conflict"

Git Stash

Lưu tạm thời changes mà chưa muốn commit:

# Lưu changes
git stash

# Xem danh sách stash
git stash list

# Apply stash gần nhất
git stash apply

# Apply và xóa stash
git stash pop

# Xóa stash
git stash drop

Undoing Changes

# Hoàn tác changes chưa stage
git checkout -- filename.txt

# Unstage file
git reset HEAD filename.txt

# Hoàn tác commit (giữ changes)
git reset --soft HEAD~1

# Hoàn tác commit (xóa changes)
git reset --hard HEAD~1

# Revert commit (tạo commit mới để undo)
git revert commit-hash

Gitignore

File .gitignore để ignore các file không cần track:

# .gitignore

# Node modules
node_modules/

# Environment variables
.env
.env.local

# Build files
dist/
build/

# OS files
.DS_Store
Thumbs.db

# IDE
.vscode/
.idea/

# Logs
*.log

GitHub Features

Pull Requests

  • Đề xuất thay đổi code
  • Code review từ team members
  • Discussion về implementation
  • Tự động chạy tests (CI/CD)

Issues

  • Track bugs và features
  • Assign cho team members
  • Sử dụng labels để phân loại
  • Link với Pull Requests

GitHub Actions

  • CI/CD automation
  • Tự động test và deploy
  • Chạy workflows khi có events

Git Best Practices

  1. Commit thường xuyên: Commit nhỏ, rõ ràng, dễ review
  2. Viết commit message tốt: Mô tả rõ ràng những gì đã thay đổi
  3. Sử dụng branches: Mỗi feature/bugfix một branch riêng
  4. Pull thường xuyên: Update code từ remote để tránh conflicts
  5. Review code: Sử dụng Pull Requests để review
  6. Không commit sensitive data: Sử dụng .gitignore
  7. Backup thường xuyên: Push lên remote repository

Commit Message Convention

# Format
type(scope): subject

# Types
feat: New feature
fix: Bug fix
docs: Documentation
style: Formatting
refactor: Code restructure
test: Add tests
chore: Maintenance

# Examples
feat(auth): add login functionality
fix(ui): resolve button alignment issue
docs: update README with setup instructions

Kết Luận

Git và GitHub là công cụ không thể thiếu cho mọi lập trình viên. Dù làm việc một mình hay trong team, Git giúp quản lý code hiệu quả, track changes và collaboration dễ dàng. Hãy thực hành Git mỗi ngày để thành thạo!
